# P0.8 — Importación por Puesto (Peoneta/Conductor) + Envío masivo por batch

## Objetivo
Dejar operación rápida para `Envio Rápido` separando candidatos por **puesto** y permitiendo importar + enviar plantilla inicial en 1–2 clicks, con trazabilidad completa.

## Formatos aceptados de importación
- `CSV`
- `XLSX`

## Headers flexibles soportados
- Teléfono: `telefono`, `teléfono`, `phone`, `celular`, `whatsapp`, `numero`, `número`
- Nombre: `nombre`, `name`, `postulante`, `candidato`
- Rol/Puesto: `rol`, `puesto`, `cargo`, `job`, `position`
- Canal: `canal`, `source`, `fuente`, `origen`
- Comuna/Ciudad: `comuna`, `ciudad`, `city`
- Estado inicial: `estado`, `status`, `etapa`, `stage`

## Reglas de normalización y dedupe
- Teléfono se normaliza a E.164 Chile (`+569...`).
- Dedupe real por E.164 (ej. `569...` y `+569...` se consideran el mismo).
- Si falta teléfono o es inválido, la fila queda en errores del resumen.

## Mapping de puesto → Program
- Si rol/puesto contiene `peoneta`/`ayudante`/`cargador` => **PEONETA**.
- Si contiene `conductor`/`chofer`/`driver`/`reparto` => **CONDUCTOR**.
- Default si no se detecta: **CONDUCTOR**.

Asignación de Program:
- `PEONETA` => Program de peonetas activo del workspace.
- `CONDUCTOR` => Program de conductores activo del workspace.

## Flujo UI (Configuración → Candidatos)
1. Subir archivo CSV/XLSX.
2. (Opcional) usar **Override puesto** para forzar todos los registros a CONDUCTOR o PEONETA.
3. Importar.
4. Revisar resumen: creados, actualizados, dedupe/omitidos, errores.
5. Acciones post-import:
   - **Abrir Inbox filtrado por este batch**
   - **Enviar plantilla inicial a todos (batch)**

## Envío masivo por batch (manual y seguro)
El envío masivo es una acción **manual auditada**. No hay envío automático al importar.

### Paso 1: Dry-run
- Calcula:
  - Total elegibles
  - Saltados (y razón)
  - Template por rol
  - Costo estimado (si pricing está configurado)

### Paso 2: Confirmación doble
- Requiere escribir `CONFIRMAR`.
- Si cambió el dry-run (`dryRunHash`), exige refrescar preview antes de ejecutar.

### Defaults de plantilla por puesto
- `CONDUCTOR` => `enviorapido_postulacion_inicio_v1`
- `PEONETA` => `enviorapido_postulacion_general_v1`

## Qué se registra en logs y estado
- Se crea `OutboundMessageLog` por cada envío masivo.
- Se guarda `templateName` real, `blockedReason` (si aplica), `dedupeKey`.
- En éxito con plantilla inicial:
  - `NUEVO -> CONTACTADO` (derivado)
  - Stage `NEW_INTAKE -> SCREENING` cuando corresponde.

## Filtros de operación
### Configuración → Candidatos
- Filtro por estado candidato
- Filtro por `Puesto` (CONDUCTOR/PEONETA)
- Filtro por Program
- Filtro por `importBatchId`

### Inbox
- Pills rápidas: `Todos`, `Conductores`, `Peonetas`
- Filtro por batch importado (desde acción post-import)
- Badge de puesto/program en la lista

## Notas
- Todo el flujo P0.8 está pensado para operar sin tocar chats históricos ni hacer envíos silenciosos.
- Pruebas automáticas recomendadas: escenarios `import_peoneta_batch_no_send` y `bulk_template_batch_send_null` (NullTransport).
